Documento de Contexto: Treatment Plan Builder
Este documento resume a evolução, funcionalidades e alterações realizadas no projeto Treatment Plan Builder para a Well Adjusted Chiropractic.

📋 Visão Geral do Projeto
O projeto consiste em uma ferramenta interativa web para criação de planos de tratamento quiroprático. Ele permite que os profissionais selecionem planos específicos, calculem custos automaticamente, adicionem recomendações extras e gerem um PDF profissional para o paciente, além de sincronizar os dados com o CRM GoHighLevel (GHL).

🔥 Funcionalidades Principais
Seleção de Planos: Suporte a planos de 16 semanas, 26 semanas, planos de descompressão espinal e planos familiares.
Cálculo Automático de Preços: Sistema dinâmico que calcula o custo total, parcelas mensais e descontos baseados no tipo de plano.
Recomendações Adicionais: Interface para adicionar até 3 serviços ou produtos extras (órteses, massagens, SoftWave, etc.).
Assinatura Digital: Integração com a biblioteca SignaturePad para que o paciente assine o plano diretamente na tela.
Geração de PDF: Uso de jsPDF e html2canvas para criar um resumo visual do plano pronto para impressão ou salvamento.
Integração GHL: Botão "Submit to GHL" que envia todos os dados do formulário para um Webhook do GoHighLevel, permitindo automação de contatos e oportunidades.
🛠️ Pilha Tecnológica
Frontend: HTML5, CSS3 (Vanilla), JavaScript (Vanilla).
Bibliotecas Externas:
SignaturePad: Captura de assinatura.
jsPDF & html2canvas: Geração de PDF a partir do DOM.
Fontes do Google: Inter, Roboto, etc.
🔄 Resumo das Alterações Recentes
1. Central de Controle de Preços (Pricing Control Center)
Implementamos uma seção no código (por volta da linha 620) que centraliza todos os preços e configurações. Isso permite que a clínica atualize valores sem precisar alterar a lógica de cálculo.
Diferenciação entre taxas de plano normal, plano familiar e "pay-as-you-go".
2. Listagem de Provedores e Unidades
Adição de dropdowns para seleção de Location (Loveland, Fort Collins, etc.) e Provider.
A lista de provedores foi atualizada recentemente com os nomes oficiais da clínica.
3. Integração com GHL (Webhook)
Configuramos o envio de dados via x-www-form-urlencoded para evitar problemas de CORS e garantir que o GHL receba todas as variáveis corretamente.
Mapeamento detalhado de campos personalizados (Custom Fields) como total_plan_cost, monthly_payment, schedule, etc.
4. Melhorias no Layout e PDF
O layout foi otimizado para ser responsivo e "limpo" (estética premium).
Ajustes finos no CSS para garantir que o PDF capture corretamente o cabeçalho e as notas, ocultando elementos desnecessários da interface (como botões).
🚀 Como Manter o Projeto
Para alterar preços: Edite a constante PRICES no arquivo 
TreatmentPlans.html
.
Para alterar provedores: Edite a constante PROVIDERS.
Para atualizar o Webhook: Altere a constante GHL_WEBHOOK_URL na seção de script.
